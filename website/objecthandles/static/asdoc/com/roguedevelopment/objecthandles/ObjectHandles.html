The default handle class to use. SpriteHandle is good for Flex 3 based applications, or Flex 4 applications where the object handles live inside a Canvas. ObjectHandles()Constructor
public function ObjectHandles(container:Sprite, selectionManager:ObjectHandlesSelectionManager = null, handleFactory:IFactory = null, childManager:IChildManager = null)

Parameters
container:Sprite — The base container that all of the objects and the handles will be added to.
 
selectionManager:ObjectHandlesSelectionManager (default = null) — A manager class that deals with which object(s) are currently selected. If you pass null, a new one will be created for you. It's often times useful to share a single selection manage across an entire app so that "selection" is global.
 
handleFactory:IFactory (default = null) — A factory capable of creating IHandle objects. If you pass null, a generic Flex 3 based factory will be created. Use the Flex4HandleFactory to make Group compatible handles.
 
childManager:IChildManager (default = null) — Flex 3 and Flex 4 applications manage children addition/removal differently (addChild vs. addElement) so I've abstracted out that functionality into an IChildManager interface that understands those differences. In general, use Flex3ChildManager for Flex3 applications and use Flex4ChildManager for Flex 4 based applications.

addDefaultConstraint()method
public function addDefaultConstraint(constraint:IConstraint):void

A constraint limits the way something could be sized or moved. For instance, you could have a minimum size. There are 3 types of constraints. A constraint that applies to a single component (set in the registerComponent call) A constraint that applies to groups of components during multi selection (set in addMultiSelectConstraint) Constraints that apply to all individual components This method adds constraints to that last one.

applyAnchorPoint()method 
protected function applyAnchorPoint(original:DragGeometry, translation:DragGeometry, currentDragRole:uint):void

When resizing, there should be an "anchor point" that doesn't move. Sometimes, we need to move the entire object around so that anchor point doesn't move. Example: Resizing a rectangle larger to the left. The width should increase and the whole thing should move to the left so the right side stays stationary. This method applies an x/y translation based upon a width/height translation and drag role

getAnchorProportion()method 
protected function getAnchorProportion(resizeHandleRole:uint):Point

Figure out which point is the anchor, and then return what the width/height proportion of the translation applies to it. Anchor = This is the point that shouldn't move. I'm only going to worry about the 8 main places a handle could be, so this will get a bit weird if you have custom handles in odd places. If that's the case for you, subclass this class and override getAnchorProportion

public function registerComponent(dataModel:Object, visualDisplay:IEventDispatcher, handleDescriptions:Array = null, captureKeyEvents:Boolean = true, customConstraints:Array = null):void

Registers a component with the ObjectHandle manager.

Parameters

dataModel:Object — The data model that represents this object. This is where values will be commited to. It should have a getter/setter for x,y,width, height, and optionally rotation.
 
visualDisplay:IEventDispatcher — This is the actual on-screen display of the object. We never set the coordinates of this object explictly, it should be bound to your data model.
 
handleDescriptions:Array (default = null) — If you want non-standard handles, create a list of HandleDescription objects and pass them in.
 
captureKeyEvents:Boolean (default = true) — Should we add event listeners to support keyboard navigation?
